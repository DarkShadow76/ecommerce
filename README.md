# Examen FullStack 2024-V-01

## Descripción del Proyecto

Crea una aplicación de gestión de pedidos para un sistema de comercio electrónico. La aplicación debe permitir crear, leer, actualizar y eliminar pedidos, así como listar todos los pedidos y obtener detalles de un pedido específico. Un pedido tiene un cliente, una lista de productos y un total. Los productos y clientes deben almacenarse en bases de datos relacionales, mientras que los pedidos deben almacenarse en una base de datos no relacional.

## Requisitos Técnicos

*   Base de Datos Relacional (PostgreSQL)
*   Base de Datos No Relacional (MongoDB)
*   Java Spring Boot
*   Webflux
*   Angular 15 o superior

## Consideraciones

1.  Las operaciones de creación y actualización de pedidos deben ser transaccionales y asegurar consistencia entre ambas bases de datos.
2.  Patrones de Diseño
3.  Implementar la aplicación de forma reactiva utilizando Spring WebFlux.
4.  Buenas Prácticas
5.  Manejar errores de forma adecuada.
6.  Validar entradas de usuario.
7.  Escribir tests unitarios para el código desarrollado.
8.  Docker

De no culminar la prueba enviarlo hasta donde este estable o crea conveniente que esta correcto su avance.
